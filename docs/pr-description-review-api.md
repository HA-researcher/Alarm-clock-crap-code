## 概要

Day6のタスク「AIコードレビュー機能（合否判定API）およびUI連携」の実装を完了しました。ユーザーがエディタで修正したコードを提出し、Gemini APIを用いてレビュー結果を受け取り、その結果に応じて画面状態（クリアや再鳴動）を切り替える一連のフローを含みます。

### 実装内容

1. **コードレビュー用APIエンドポイントの作成 (`app/api/review-code/route.ts`)**
   - 太田さんが作成したプロンプト仕様(`ai-prompts.md`)に基づき、元のクソコードとユーザーの修正コード等のパラメータを受け取ってレビューを実行するAPIを実装しました。
   - `gemini-1.5-flash` モデルと公式SDK(`@google/genai`)を使用してJSONでレスポンスを返します。
2. **フロントエンド用サービスクラスの作成 (`lib/alarm/CodeReviewService.ts`)**
   - 新しく作成したAPIを呼び出す（リクエストを送信する）ためのラッパー関数 `reviewCode` と、レスポンスの型定義（`ReviewResult`, `ReviewCodeParams`等）を実装しました。
3. **エディタ画面の提出・UI連携 (`app/challenge/page.tsx` & `components/challenge/SubmissionArea.tsx`)**
   - 「提出する」ボタン押下時に `CodeReviewService` を呼び出し、結果待ちの間はボタンをディセーブルにしてローディング状態を表示するように改修しました。
   - AIのレビュー結果（合否メッセージや差分フィードバック）を画面左側のAIアシスタントパネルに表示するロジックを追加しました。
4. **状態遷移管理の更新 (`lib/alarm/flow.ts`)**
   - レビュー実行中を表す `reviewing` ステータスと、強制停止時の `force_stopped` ステータスを `AppState` に追加し、整合性を保って状態遷移が行えるよう更新しました。

### 確認・テスト手順

1. `npm install` または `npm ci` で最新の依存関係（最新SDKや追加した `lucide-react` アイコン等）をインストールし、`npm run dev` で起動します。
2. `/challenge` 画面で適当なコードに編集後、「パッチを提出」ボタンを押してください。
3. レビュー中ステータスを経て左側のパネルに合否判定のテキストが表示され、内部の `AppState` が適切に遷移している（合格なら `cleared`、不合格なら `alarming`）ことを動作確認してください。
